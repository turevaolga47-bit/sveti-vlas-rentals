## Часть 1. 5 примеров Telegram-решений для посуточной аренды квартир

> Важный вывод: полноценных Telegram Mini App именно для посуточной аренды почти нет — рынок занят ботами-уведомителями и Telegram-каналами. Это и есть наша возможность.

---

### 1. @sutochnoru_bot — Sutochno.ru (Россия, 400 000+ объявлений)

**Что делает:**  
Бот-уведомитель крупнейшего российского сервиса посуточной аренды. Подключает аккаунт sutochno.ru к Telegram — новые запросы от гостей, уведомления об оплате и сообщения приходят в Telegram вместо email. Сама платформа полноценная: мгновенное бронирование, кешбэк до 30%, приложение для арендодателей «Сдавайся».

**Сильные стороны:**
- Крупнейшая база в России (400 000+ объектов, 13 стран)
- Нет комиссии с гостей — выгоднее Booking и Авито
- Мгновенное бронирование без подтверждения хозяина
- Кешбэк-программа удерживает повторных гостей
- Telegram-бот держит арендодателей «на связи» без входа в приложение

**Слабые стороны:**
- Бот — **только труба для уведомлений**, поиск и бронирование происходят на сайте
- Нельзя найти квартиру, посмотреть фото или забронировать прямо в Telegram
- Частые жалобы (irecommend.ru): хозяева отменяют после предоплаты, иногда поднимая цену
- Залоги снимают без объяснений, поддержка медленная и шаблонная
- Фото на сайте часто не соответствуют реальности

**Что можно сделать лучше:**  
Встроить поиск, галерею, календарь доступности и оплату прямо в TMA — полный путь от «хочу снять» до «забронировано» без выхода из Telegram.

### 2. @re_kufar_bot — Kufar Недвижимость (Беларусь)

**Что делает:**  
Telegram-бот крупнейшей доски объявлений Беларуси. Один из немногих ботов, который явно поддерживает **краткосрочную/посуточную аренду** как отдельный тип. Пользователь выбирает тип аренды, город, фильтры (тип жилья, комнаты, цена, количество гостей) — и бот присылает новые подходящие объявления из базы Kufar.

**Сильные стороны:**
- Посуточная аренда — именной отдельный режим, не просто фильтр
- Фильтр по количеству гостей — редкость для Telegram-ботов
- Проактивные push-уведомления: настроил один раз, получаешь новинки
- Реальная база верифицированных объявлений
- Бесплатный

**Слабые стороны:**
- Только Беларусь
- Бот отдаёт ссылку на сайт Kufar — всё просматривается и бронируется там, не в Telegram
- Фото в чате не показываются — нужно переходить по ссылке
- Нет проверки доступности на конкретные даты внутри бота
- Нет оплаты в Telegram
- После нахождения жилья бот становится бесполезным — нет постаренды функций

**Что можно сделать лучше:**  
Показывать карусель фото прямо в Mini App, позволять выбрать даты и видеть доступность, не уходя на сайт. Завершать бронирование с оплатой внутри Telegram.

### 3. GuestiBot (`guestibot.ru`) — бот для управления гостями

**Что делает:**  
Единственный инструмент в исследовании, созданный **специально для посуточной аренды**. Но работает со стороны арендодателя, а не гостя. После бронирования (через Sutochno, Авито, любой канал) гость получает доступ к боту: автоматический чекин с кодами доступа и WiFi, меню услуг (уборка, смена белья, продление), документальная верификация гостя, чекаут, сбор отзыва. Цена: от 80 ₽/мес за объект.

**Сильные стороны:**
- Создан именно для посуточной аренды — единственный такой в нише
- Решает главную боль хозяев: бесконтактный заезд без личной встречи
- Автоматизирует однотипные сообщения (WiFi, правила, парковка) — экономит часы
- Очень дёшево (меньше $1/мес за объект)
- Работает полностью в Telegram — гостю не нужно ничего скачивать

**Слабые стороны:**
- **Нет каталога и поиска** — гости не находят жильё через GuestiBot
- Нет оплаты через бота — приём денег идёт через сторонние платформы
- Нет синхронизации с Avito/Sutochno-календарями
- Отзывы, собранные в боте, не выгружаются на публичные платформы
- Закрывает только середину пути (чекин/пребывание/чекаут), не начало и конец

**Что можно сделать лучше:**  
Объединить функции GuestiBot (операционная часть) с каталогом и бронированием в одном TMA — гость находит квартиру, бронирует, платит, получает код доступа — всё в одном приложении без переключений.

### 4. Avito Путешествия — крупнейший агрегатор посуточной аренды в России

**Что делает:**  
Запущен в апреле 2024, перезапуск раздела посуточной аренды Авито. 300 000+ объектов от 1 дня до 1 месяца. Онлайн-бронирование с обязательной предоплатой (30% по умолчанию), мгновенное бронирование, гарантия безопасной оплаты, кешбэк 5%, верифицированные объекты. Никакой Telegram-интеграции нет.

**Сильные стороны:**
- Крупнейшая база (~300 000 объектов), охват малых городов
- Бренд Авито — доверие от массовой аудитории
- Мгновенное бронирование для последнего момента
- Прозрачная цена (показывается до оплаты)
- Хорошие инструменты для арендодателей

**Слабые стороны** (рейтинг otzovik.ru: 1.2/5, только 4% рекомендуют):
- Комиссия 15–20% с хозяев + сервисный сбор с гостей
- Хозяин **не может отказать гостю** — принудительное принятие бронирований
- Поддержка медленная, отвечает шаблонами, споры не решаются
- Жалобы на хозяев, повышающих цену после подтверждения
- Баги в календаре, блокирующие объявления без причины
- Полное отсутствие Telegram — вся коммуникация в закрытой экосистеме Авито

**Что можно сделать лучше:**  
TMA может дать хозяину право выбора гостя (просмотр профиля и рейтинга до подтверждения), прямое общение в Telegram, прозрачную комиссию с первого экрана и реальное разрешение споров через чат-медиацию — а не через закрытый тикет.

### 5. Telegram-каналы посуточной аренды (@Kvartiru_Posutochno и аналоги)

**Что делает:**  
Децентрализованная экосистема: каналы и чаты, где хозяева публикуют объявления о посуточной аренде, а туристы спрашивают рекомендации. Примеры: @Kvartiru_Posutochno (канал КВАРТИРЫ-ПОСУТОЧНО.РФ), болгарские каналы типа «Аренда жилья Болгария» (1 177 подписчиков), десятки локальных групп по городам и курортам. Нулевая комиссия — хозяин и гость договариваются напрямую через DM.

**Сильные стороны:**
- Нулевая комиссия — ни платформа, ни агрегатор не берут процент
- Прямой контакт: реальный хозяин отвечает лично, формируется доверие
- Покрывают курортные направления, которых нет на крупных платформах
- Работают полностью в Telegram — не нужно ничего устанавливать
- Местные советы и инсайды (лучшие кварталы, как не попасть на мошенника)

**Слабые стороны:**
- Нет структурированного поиска — листаешь посты вручную или пишешь в чат
- Нет проверки доступности: хозяин пишет «свободно с X по Y», но без живого календаря
- Нет защиты платежей — предоплата переводом, без эскроу
- Нет стандарта объявлений — качество, формат и количество фото у каждого своё
- Нет системы рейтингов — доверие строится на истории переписки
- Открытые каналы — лакомое место для мошенников под видом арендодателей

**Что можно сделать лучше:**  
TMA сохраняет всё лучшее из канального подхода (прямой контакт, нулевая/низкая комиссия) и добавляет структуру: стандартизированные карточки с фото, живой календарь, верифицированные профили хозяев, эскроу-платёж и рейтинги после каждого заезда.

## Часть 2. Типичные экраны и обязательные элементы

### Обязательные экраны (must-have)

| Экран | Назначение | Ключевые элементы |
|---|---|---|
| **Сплэш / Загрузка** | Закрывает задержку первого запроса | Skeleton UI в стиле приложения, без пустого белого экрана |
| **Главная / Каталог** | Основная точка входа для просмотра | Поиск, чипы-фильтры, карточки объектов (сетка или список), sticky-шапка |
| **Детальная страница** | Полная информация об объекте | Фотогалерея, заголовок, цена/статус, описание, характеристики, MainButton «Забронировать» |
| **Поиск и фильтры** | Сужение каталога | Поле поиска, сегментированный выбор, результаты с количеством |
| **Форма бронирования** | Сбор деталей резервации | Выбор дат, количество гостей, контактные данные, спецпожелания |
| **Итоговый расчёт** | Ревью перед оплатой | Разбивка: ночи × цена + уборка + залог = итого, промокод |
| **Оплата** | Чекаут | Telegram Payments native sheet через `openInvoice` |
| **Подтверждение** | Пост-бронирование | Lottie-анимация, номер брони, детали, «Поделиться» / «Мои брони» |
| **Профиль / Мои бронирования** | Личный кабинет | Аватар из Telegram, список прошлых и предстоящих броней |
| **Пустое состояние / Ошибка** | Graceful failure | Иллюстрация, читаемое сообщение, кнопка восстановления |

### Рекомендуемые дополнительные экраны

| Экран | Назначение |
|---|---|
| **Карта** | Позиция объектов относительно пляжа/марины |
| **Категории** | Если есть 2+ типа объектов (студии / 1BR / 2BR) |
| **Избранное** | Сохранённые объекты + уведомления о цене |
| **Отзывы** | Звёздный рейтинг + текст, повышение доверия |
| **Онбординг (1–3 слайда)** | Только если сервис неочевиден — не перегружать |
| **Настройки** | Язык, уведомления — через SettingsButton (шестерня) |

### Обязательные Telegram-нативные элементы

| Элемент | Когда использовать |
|---|---|
| **MainButton** | Единственное главное действие на экране: «Забронировать», «Оплатить», «Далее» |
| **BackButton** | Показывать на всех экранах кроме главного, скрывать на корне |
| **ClosingConfirmation** | Включать с шага 1 форм бронирования — защита от случайного закрытия |
| **HapticFeedback** | `light` — нажатие кнопок; `success` — подтверждение брони; `error` — ошибка оплаты |
| **showPopup / showConfirm** | Для необратимых действий (отмена брони) |
| **SettingsButton** | Для настроек — не занимает место в UI |

Никаких хардкода цветов — только Telegram CSS-переменные:

## Часть 4. Что мы делаем лучше всех

### Уровень 1 — День 1, ключевые дифференциаторы

**1. Fullscreen фотогалерея**  
TMA 2.0 fullscreen mode (`web_app_request_fullscreen`). 10–20 фото на объект с метками комнат. Отключение вертикального свайпа во время галереи через `web_app_setup_swipe_behavior` — не даём случайно закрыть приложение. Это уже обгоняет 100% существующих Telegram-решений для аренды.

**2. Живой календарь занятости + запрос на бронирование**  
Месяц-вью: занятые даты красным, свободные — зелёным. Цена за ночь прямо на ячейке календаря (для сезонного ценообразования). В v1 — «Запросить эти даты» с преднаполненным сообщением. Это уже кардинально лучше любого Telegram-канала.

**3. Прозрачный итоговый расчёт**  
Цена × ночи + уборка + залог = итого. Никаких «от 30 до 55 EUR/ночь» без расшифровки. Это сигнал доверия, которого не хватает всему рынку.

**4. Слой доверия**  
- Бейдж верифицированного владельца  
- Количество завершённых аренд  
- Кнопка «Позвонить / WhatsApp» — реальный человек, не бот  
- Дата последнего обновления фото («Фото обновлены: июнь 2025»)

**5. Deep-link на каждый объект**  
`https://t.me/YourBot/appname?startapp=apt_123` — при пересылке в чат друзьям открывается конкретная квартира, не главная страница. Вирусное распространение в группах туристов.

### Уровень 2 — Сезон 1

**6. Визуальные фильтры (не slash-команды)**  
Bottom sheet: слайдер цены (€20–€200), количество комнат, чипы удобств (бассейн, вид на море, кондиционер, парковка, рядом с пляжем). Строка активных фильтров над списком.

**7. Избранное + уведомления**  
Лайк на объект → бот пишет, когда цена снизилась или открылись нужные даты. Главный механизм удержания для сезонного продукта: закладывают мечту в марте, получают пинг в апреле.

**8. Карта с геолокацией**  
TMA 2.0 Geolocation API — показываем объекты на карте относительно пляжа/марины. «2 минуты до моря» превращается из текстового утверждения в отметку на карте.

**9. Поделиться с преднаполненным сообщением**  
Кнопка «Поделиться» — генерирует пост: фото + название + цена + даты + deep-link. Один тап — и форматированная карточка объекта летит в любой чат. Так e-commerce Mini Apps получают 60% новых пользователей без рекламы.

**10. Мультиязычность с первого дня**  
🇷🇺 Русский / 🇧🇬 Болгарский / 🇬🇧 Английский. Язык определяется автоматически из настроек Telegram — не нужен переключатель.

### Уровень 3 — Сезон 2

**11. AI-поиск на естественном языке**  
«Квартира с 2 спальнями рядом с мариной, бюджет €80/ночь, 10–24 июля, 2 взрослых + ребёнок» → автоматически применяются фильтры. Паттерн уже реализован в n8n Airbnb Telegram Agent.

**12. Кабинет владельца**  
Владелец управляет занятостью, загружает фото с телефона через TMA 2.0 Camera API, получает уведомления о новых запросах. Это решает главную боль любого каталога — актуальность данных.

## Часть 5. Известные проблемы платформы (учесть при разработке)

| Проблема | Решение |
|---|---|
| Горизонтальный свайп в галерее закрывает приложение на iOS | Отступы 10–16px по краям + `setup_swipe_behavior` |
| Клавиатура перекрывает поля ввода на iOS | Использовать `--tg-viewport-stable-height` для кнопок |
| Белый мелькающий экран при запуске | `Telegram.WebApp.ready()` немедленно + skeleton UI |
| Дублирование навигации (свой back + Telegram BackButton) | Только нативный BackButton, без своей кнопки «назад» |
| Недоверие к незнакомому интерфейсу внутри Telegram | Верифицированные владельцы, реальные фото с датами, tap-to-call |
| Потеря данных при обрыве связи | `ClosingConfirmation` + сохранение состояния формы локально |

## Экспертная оценка

> Идея: Telegram Mini App — каталог квартир посуточной аренды в Святом Власе, Болгария.  
> Пользователь открывает ТГ-бота → нажимает кнопку → видит квартиры с фото, ценами и кнопкой «Забронировать».  
> Оценка с трёх точек зрения: UX-дизайнер / Маркетолог / Telegram-разработчик.

### UX-дизайнер: ошибки, обязательные элементы, что бесит

#### Типичные ошибки новичков

**Архитектурные:**
- Делают один экран «список квартир» и считают это каталогом. Нет состояния «пусто», нет скелетона, нет обработки ошибки — пользователь видит белый экран и думает, что приложение сломалось
- Копируют структуру веб-сайта внутрь TMA: огромные фильтры сверху, таблица цен, футер с контактами. TMA — не мобильный сайт, у него жёсткие ограничения по вертикальному пространству
- Не используют нативные компоненты (MainButton, BackButton, HapticFeedback) — приложение ощущается как чужеродный iframe

**Ошибки каталога аренды:**
- Показывают цену без периода: «2500 ₽» — за ночь? За неделю? За всё? Пользователь уходит, не разобравшись
- Карточка без ключевых параметров: нет этажа, нет вида, нет расстояния до моря — три параметра, которые смотрят в первую очередь
- Форма бронирования — один длинный скролл с 12 полями сразу. Пользователь закрывает приложение

**Ошибки с датами:**
- Два отдельных поля через нативный `<input type="date">` — на iOS выглядит как мусор. Нужен кастомный range-picker
- Не блокируют занятые даты — пользователь выбирает даты, жмёт «Забронировать», получает ошибку «недоступно». Это слом доверия

#### Что обязательно должно быть

- **Цена за ночь крупно + итог сразу.** Пользователь вводит даты → приложение мгновенно показывает «7 ночей × 2500 ₽ = 17 500 ₽»
- **Фото с горизонтальным свайпом прямо в карточке каталога** — не только на детальной странице. Пользователи не открывают каждую квартиру
- **Фильтр «свободно в мои даты» как первое действие** — не после просмотра 20 квартир, а до
- **WhatsApp/Telegram-контакт хозяина одной кнопкой** — аудитория 35+ не доверяет онлайн-оплате без живого человека
- **Статус бронирования в реальном времени** + уведомление в чат при смене статуса
- **Скелетон-загрузка** на каталоге — не спиннер, а скелетон: снижает воспринимаемое время ожидания
- **Только нативный BackButton**, без кастомной стрелки в UI

#### Что бесит пользователей

- **Галерея, закрывающая приложение на iOS** — горизонтальный свайп = iOS закрывает TMA. Всё выбранное теряется. Пользователь не возвращается
- **Клавиатура перекрывает поля формы на iOS** — вводит имя, клавиатура закрывает поле «телефон» ниже. Нет автоскролла = нет заполненной формы
- **Карточки без цены** — заставляют открывать каждую квартиру. После 3–4 таких карточек — усталость и уход
- **«Белая вспышка» при запуске** — разрыв при тёмной теме Telegram. Решается через `background_color` в манифесте
- **Сброс каталога в начало** после нажатия «назад» с детальной страницы — нужно листать заново. Это физически больно
- **Нет сообщения об ошибке при обрыве связи** — особенно критично в Болгарии на роуминге

#### Конкретные UX-рекомендации

1. **Онбординг = выбор дат, а не обучение продукту.** Первый экран — два поля: «Когда приезжаете?» и «Когда уезжаете?» + кнопка «Показать квартиры». Конверсия в бронирование растёт кратно
2. **Галерея как отдельный слой**, а не отдельный роут. `setSwipeBehavior({ allow_vertical_closing: false })` + отступы 16px по краям. При закрытии пользователь остаётся на детальной странице
3. **Трёхшаговая форма:** Шаг 1: Даты + гости → Шаг 2: Имя + телефон → Шаг 3: Итог. MainButton меняет текст: «Далее» → «К оплате» → «Забронировать». Не больше 3 полей одновременно
4. **Жёсткий минимум карточки:** фото (свайпабельные, 2–3 штуки), тип («Студия у моря»), расстояние до моря в метрах, спальных мест, цена за ночь, бейдж доступности. Если карточка требует скролла — она сломана
5. **Восстановление состояния** — сохранять в `localStorage` даты, фильтры и позицию скролла. Пользователь случайно закрыл — при открытии восстанавливаем с тостом «Продолжаем с того места»
6. **Блок доверия перед оплатой** — фото хозяина + имя + «на связи в Telegram» + кнопка написать напрямую + политика отмены одной строкой. Без этого значительная часть аудитории не нажмёт «Оплатить»
7. **Push-уведомления — не «фича v2»**, а часть продукта с первого дня. Подтверждение брони, напоминание за 3 дня, напоминание в день заезда с адресом и кодом. Ради этого и делается TMA

### Маркетолог: что заставит забронировать, конверсия, страхи

#### Главный психологический триггер

Русскоязычный турист боится не Болгарии — он боится **заплатить и не получить ключи**. Опыт с Booking (отмены), Авито (скамы) и серыми каналами сформировал реальный паттерн недоверия. Поэтому:

- **«Видел это место у знакомых»** — если квартира пришла шерингом из семейного чата, порог доверия падает в 2–3 раза. Это сильнейший конвертер
- **Видеть реального человека** — верифицированный профиль хозяина с фото конвертирует лучше любых бейджей. Аудитория покупает «у Лены из Бургаса», не у «Апартамент №47»
- **Цена без сюрпризов** — прозрачный breakdown убивает главную боль Авито и Airbnb
- **Живой календарь с занятыми датами** — доказывает реальность объекта. Пустой календарь = фейк

#### Элементы, повышающие конверсию

**Карточка квартиры:**
- Первое фото — вид на море или бассейн. Не интерьер. Продаётся мечта, не метраж
- Галерея минимум 8–12 фото: кухня, ванная, вид из окна, парковка
- Расстояние до моря **в минутах пешком**, не в метрах: «7 минут» > «550 м»
- Вместимость: «Подходит для семьи с 2 детьми» > «4 спальных места»
- Топ-3 фичи иконками: Wi-Fi / Кондиционер / Парковка — без них не читают дальше
- Счётчик бронирований: «Забронировали 23 раза этим летом»
- Последний отзыв с датой и реальным именем прямо на карточке

**CTA — текст кнопки:**

| ❌ Не использовать | ✅ Использовать |
|---|---|
| «Забронировать» — звучит как риск | «Узнать доступность» — нулевое обязательство |
| «Купить» — слишком финально | «Проверить даты» → «Закрепить за собой» |
| «Оформить заявку» — бюрократично | «Подтвердить бронь за [СУММА]» — конкретная цифра в кнопке |

**Траст-сигналы:**
- Российский/украинский/израильский номер хозяина — «свой» хозяин важнее рейтинга
- Способы оплаты крупно: «Карта РФ / Крипта / Наличные при заезде» (особенно важно при санкционных проблемах)
- Политика отмены простым языком: «Отмена за 7 дней — полный возврат» без звёздочек
- Кнопка «Написать хозяину» **до брони** — прямой ответ на боль Авито, где контакт заблокирован
- Год последнего ремонта — снимает страх «убитой квартиры»

#### Что убивает конверсию — страхи аудитории

| Страх | Нейтрализация |
|---|---|
| «Квартиры не существует» | Видео от хозяина, геолокация на карте, дата последнего обновления |
| «Деньги возьмут, а при заезде что-то не так» | Чёткая политика возврата прямо в карточке |
| «Не смогу связаться при проблеме» | Telegram-аккаунт хозяина виден до оплаты |
| «Как иностранца обманут» | Интерфейс на русском, хозяин говорит по-русски — отметить явно |
| «Зачем бронировать сейчас» | Реальный дефицит на календаре + «78% июля уже занято» |

#### Конкретные маркетинговые рекомендации

1. **Первый экран продаёт эмоцию, не функцию.** Hero-изображение с морем + «Святой Влас — квартиры у моря. Июль–август 2026» + кнопка «Смотреть свободные». Список квартир — это работа. Море — мечта. Продавать мечту первые 3 секунды
2. **Верификационный бейдж с человеческим лицом.** «Проверено командой [фото + имя]» — живой человек, осмотревший квартиру. Работает в постсоветском контексте лучше алгоритмических рейтингов
3. **Share-механика с готовым текстом.** Кнопка «Поделиться» генерирует: *«Смотри какую квартиру нашла на август — вид на море, 3 спальни, 4500 ₽/ночь. Вот ссылка прямо на неё: [deeplink]»*. Получатель открывает диплинк → видит «Вам поделился(ась) [имя]». Холодный трафик превращается в тёплый. Цель: 40%+ бронирований через шеринг
4. **Ценовой якорь против Airbnb.** На карточке: «На Airbnb эта квартира стоила бы ~6 800 ₽/ночь. Здесь: 4 500 ₽/ночь. Без комиссий.» Обновлять ежемесячно — квартир немного, это не накладно
5. **Честная срочность — реальные данные, не фейк.** «Июль 2026: 78% квартир уже забронировано» — агрегированная цифра. Подсвеченный календарь с занятыми датами — визуально показывает дефицит без слов. Честная срочность работает, фейковые таймеры разрушают доверие
6. **Воронка через бота до открытия TMA.** Пользователь пишет боту «хочу квартиру» → 3 вопроса (даты / количество человек / бюджет) → «Нашёл 4 квартиры под ваш запрос» + кнопка открытия TMA с применёнными фильтрами. Пользователь видит не 20, а 4 квартиры — именно для него
7. **Виральный цикл после брони.** Экран подтверждения: «Едете с кем-то? Поделитесь — если они забронируют, получите 500 ₽ на следующую поездку.» Репост в семейные чаты происходит органически в момент радости — дать инструмент и маленький стимул

### Telegram-разработчик: ограничения, что работает, что не делать

#### Платформенные ограничения

**Галерея:**
- iOS swipe-to-close конфликт — горизонтальный свайп ближе 10px от края закрывает TMA. Нужен `setSwipeBehavior({ allow_vertical_closing: false })` и `padding: 0 16px` на touch-зонах
- Fullscreen через TMA 2.0 (`expand()` + `requestFullscreen: true` в манифесте) — но верхний тулбар Telegram остаётся видимым на Android
- 20+ фото по 2MB без lazy loading = OOM crash на бюджетных Android. Лимит WebView ~150–200MB

**Календарь:**
- Нет Server-Sent Events и WebSockets в некоторых конфигурациях WebView. Реальное обновление занятости — только polling или при открытии. Это создаёт окно для double-booking без серверного soft-hold
- `localStorage` ненадёжен между сессиями — WebView может быть убит ОС

**Платежи:**
- `openInvoice()` работает только через официальных провайдеров Telegram Payments. Болгарского эквайринга (Borica, EasyPay) в списке нет — нужен Stripe с международным счётом или болгарским юрлицом
- После оплаты нет callback с деталями транзакции — только статус `paid/cancelled/failed`. Реальное подтверждение приходит через bot webhook `successful_payment`. Архитектура **обязательно асинхронная**
- Нет сохранённых карт для повторных платежей (кроме Telegram Stars)

**Карта:**
- Google Maps недоступен без API key и CSP. Leaflet + OpenStreetMap — единственный надёжный вариант
- Геолокация `requestLocation()` недоступна в Desktop-версии Telegram

**Уведомления:**
- Нет Web Push в TMA — принципиальное ограничение. Уведомления только через бота. Пользователь обязан **сначала написать боту** (`/start`). Этот шаг нужно вшить в онбординг

**Мультиязычность:**
- `initDataUnsafe.user.language_code` ненадёжен — возвращает язык интерфейса Telegram, не язык пользователя. Русскоязычный из Германии получит `de`. Нужен явный переключатель языка в UI

#### Что работает лучше в TMA, чем на сайте

- **Аутентификация** — `initDataUnsafe.user` даёт имя и username без регистрации. `requestContact()` — телефон с согласия. Форма бронирования из 6 полей сокращается до 2. HMAC-валидация `initData` — полноценная аутентификация без JWT/OAuth
- **Платежи** — Telegram Payments конвертирует лучше внешних форм. Пользователь не покидает приложение. `answerPreCheckoutQuery` позволяет проверить доступность в момент оплаты
- **Вирусность** — `shareMessage()` + deep links: `t.me/Bot/app?startapp=apt_42` открывает TMA сразу на нужной квартире. При получении диплинка — сразу нужный объект
- **UX-интеграция** — MainButton с динамическим текстом («Проверить даты» → «Забронировать» → «Оплатить 420 лв»), HapticFeedback при выборе дат, BackButton синхронизирован с системной кнопкой Android
- **Home screen shortcut** через `addToHomeScreen()` (TMA 2.0) — иконка без App Store, обновления мгновенные

#### Что лучше не делать

- Кастомные стрелки «назад» — дублирование с BackButton API: системная кнопка Android и кнопка в UI делают разные вещи
- `framer-motion` и тяжёлые CSS-анимации на list/grid — Redmi 9 / Samsung A12 (целевой бюджетный турист) выдаёт 10–15 fps. Только `transform` и `opacity`
- `react-datepicker`, MUI DatePicker и тяжёлые date-picker библиотеки — медленно рендерятся в WebView, не адаптированы под TMA. Кастомный лёгкий календарь ~3KB
- Хранить статус бронирования только в клиенте — `invoiceClosed` может не прийти при обрыве. Статус всегда определяется сервером через bot webhook
- Оптимистичное бронирование без серверного lock — нужен soft-hold на 12 минут
- Хардкодированные цвета — только CSS-переменные `--tg-theme-*`. TMA открывается в dark/light по настройкам Telegram
- `100vh` для высоты — только `--tg-viewport-stable-height`. На iOS с клавиатурой `100vh` некорректен

#### Конкретные технические рекомендации

**1. Стек — минимальный и быстрый**
```
React 18 + Vite (не CRA, не Next.js — SSR не нужен)
Zustand (не Redux — меньше boilerplate)
@tma.js/sdk (официальная типизированная обёртка)
Leaflet 1.9 напрямую (не react-leaflet)
ky (3KB) + React Query для кэша
CSS Modules + PostCSS
Целевой bundle: < 180KB gzipped
```

**2. Защита от double-booking — трёхфазная архитектура**
```
1. SEARCH: GET /availability → список дат (кэш 60 сек, Redis)
2. HOLD: POST /hold → soft-hold на 12 минут с hold_token
         Если hold занят → 409 Conflict → «Даты заняты»
3. PAYMENT: openInvoice(url) с hold_token в payload
         pre_checkout_query → проверить hold активен
         successful_payment → создать бронирование, удалить hold
Cron: удалять просроченные holds каждые 2 минуты
```

**3. Платёжный flow — правильные состояния**
```javascript
WebApp.openInvoice(invoiceUrl, (status) => {
  if (status === 'paid') {
    // НЕ считать подтверждённым здесь!
    // Ждать successful_payment от bot webhook
    startPollingBookingStatus(holdToken); // polling каждые 2 сек, таймаут 30 сек
  }
  if (status === 'cancelled') {
    releaseHold(holdToken); // Освободить немедленно
  }
});
```

**4. Галерея без конфликта iOS swipe**
```javascript
// При открытии галереи
WebApp.setSwipeBehavior({ allow_vertical_closing: false });
// CSS: .gallery { padding: 0 16px; touch-action: pan-x; }
// При закрытии — восстановить
WebApp.setSwipeBehavior({ allow_vertical_closing: true });
```
Для листания — `swiper.js` (только модули `Navigation` + `Virtual`) или кастомный pointer handler ~2KB.

**5. Форма + клавиатура на iOS**
```css
.booking-footer {
  position: fixed;
  bottom: 0;
  padding-bottom: env(safe-area-inset-bottom);
}
.booking-form {
  height: var(--tg-viewport-stable-height); /* не 100vh ! */
  overflow-y: auto;
}
```

**6. Deep links формат**
```
https://t.me/SvetiVlasRentBot/rent?startapp=apt_42_2026-06-01_2026-06-07

const startParam = WebApp.initDataUnsafe.start_param;
// → "apt_42_2026-06-01_2026-06-07"
```

**7. Онбординг для уведомлений (единственный рабочий способ)**
```
1. TMA: кнопка «Уведомить когда освободится»
2. WebApp.openTelegramLink('https://t.me/Bot?start=watch_apt42')
3. Бот: «Слежу за апартаментом. Уведомлю как только откроются даты.»
4. Сервер записывает { telegram_user_id, apt_id, dates }
5. Cron проверяет, бот отправляет сообщение
Без шагов 2–3 уведомления технически невозможны.
```

### Общий список рекомендаций (приоритет)

#### Критично — без этого продукт провалится
1. Решить iOS swipe в галерее (`setSwipeBehavior` + отступы 16px)
2. Серверный soft-hold (12 мин) для защиты от double-booking
3. Прозрачная цена: ночи × тариф + уборка + залог = итого — прямо на карточке
4. Трёхшаговая форма бронирования, не один длинный скролл
5. `--tg-viewport-stable-height` везде вместо `100vh`

#### Важно — влияет на конверсию
6. Первый экран — выбор дат, не список всех квартир
7. CTA: «Узнать доступность» → «Закрепить за собой» → «Подтвердить бронь за [сумма]»
8. Блок доверия перед оплатой: фото хозяина + кнопка написать + политика отмены
9. Share-механика с готовым текстом и deep-link на конкретную квартиру
10. Восстановление состояния (даты, фильтры, скролл) при повторном открытии

#### Рост — второй сезон
11. Ценовой якорь против Airbnb на каждой карточке
12. AI-поиск на естественном языке
13. Кабинет владельца с управлением доступностью
14. Реферальная программа: поделился → друг забронировал → скидка
