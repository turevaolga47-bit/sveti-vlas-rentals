# Brief: Telegram Mini App — Аренда квартир, Святой Влас

> Версия: 1.0 (первый релиз)  
> Язык интерфейса: Русский  
> Платформа: Telegram Mini App 2.0  
> Дизайн: нативный Telegram (CSS-переменные, MainButton, BackButton, HapticFeedback)  
> Основан на: research.md + экспертная оценка (UX / Маркетолог / Разработчик)

---

## Концепция в одной фразе

Турист открывает бота в Telegram → за 3 нажатия видит свободные квартиры на свои даты → листает фото → бронирует и платит не выходя из Telegram.

---

## Навигация

```
НИЖНЯЯ ПАНЕЛЬ (3 таба, V1):
  [Квартиры]  [Мои брони]  [Профиль]
```

Нижняя панель видна на всех корневых экранах.  
На вложенных экранах (детали, бронирование) — скрыта. Работает нативный **BackButton**.

## Экраны V1

### Экран 0 — Сплэш / Инициализация

**Что видит пользователь:**
- Фон цвета `--tg-theme-bg-color` (мгновенно, без белой вспышки)
- Скелетон: две прямоугольные тени-заглушки (имитируют карточки квартир)
- Лого / название приложения по центру

**Технические действия (невидимы пользователю):**
- `Telegram.WebApp.ready()` — вызывается первой строкой
- `WebApp.expand()` — разворачиваем на весь экран
- Проверка `localStorage`: если есть сохранённые даты + позиция скролла → восстановить, показать тост «Продолжаем с того места»
- Проверка `start_param`: если пришёл deep-link `apt_42_...` → перейти сразу на Экран 4

**Переходы:**
- Данные загружены → **Экран 1** (первый запуск) или **Экран 2** (повторный с сохранёнными датами)

### Экран 1 — Выбор дат (первый запуск / смена дат)

> По рекомендации UX-эксперта: первый экран = выбор дат, не список квартир.  
> По рекомендации маркетолога: продаём мечту, не список.

**Что видит пользователь:**
```
┌─────────────────────────────────────┐
│  [Фоновое фото: море, пляж, Святой  │  ← hero-изображение, полная ширина
│   Влас. Полупрозрачный оверлей]     │
│                                     │
│  Святой Влас                        │  ← крупный заголовок, белый текст
│  Квартиры у моря · Лето 2026        │  ← подзаголовок
│                                     │
│  ┌─────────────┐  ┌─────────────┐   │
│  │  Заезд      │  │  Выезд      │   │  ← два поля, tap → открывает календарь
│  │  15 июля    │  │  22 июля    │   │
│  └─────────────┘  └─────────────┘   │
│                                     │
│  Сколько гостей?                    │
│  [−]  2 гостя  [+]                  │  ← степпер, кнопки 44px
│                                     │
└─────────────────────────────────────┘
       [ Показать квартиры ]           ← MainButton Telegram (синий)
```

**Элементы:**
- Hero-фото (море) — полная ширина, фиксированная высота 220px
- Заголовок + подзаголовок поверх фото
- Поле «Заезд» — tap открывает bottom-sheet с кастомным календарём
- Поле «Выезд» — появляется/активируется после выбора даты заезда
- Степпер «Количество гостей» (1–8), кнопки — и + не менее 44px
- MainButton: «Показать квартиры» — активна только когда обе даты выбраны

**Действия:**
- Тап на «Заезд» → открывается календарь (bottom sheet)
- Тап на «Выезд» → открывается календарь (bottom sheet)
- Тап [−] / [+] → меняет количество гостей, `HapticFeedback.impactOccurred('light')`
- MainButton «Показать квартиры» → **Экран 2** (каталог, отфильтрованный по датам)

**Технические детали:**
- Кастомный range-picker (НЕ `<input type="date">`)
- Занятые даты в календаре — серые, некликабельны
- MainButton.disable() пока не выбраны обе даты
- Сохранять выбранные даты в `localStorage` при каждом изменении

**Что видит пользователь:**
```
┌─────────────────────────────────────┐
│ 15–22 июля · 2 гостя  [Изменить]   │  ← sticky строка с параметрами поиска
│ Найдено: 8 квартир   [Фильтры ≡]   │  ← кол-во + кнопка фильтров
├─────────────────────────────────────┤
│ ┌─────────────────────────────────┐ │
│ │ [фото 1] ← свайп → [фото 2]    │ │  ← 2–3 фото, свайп горизонтальный
│ │                   ● ○ ○         │ │  ← индикатор фото
│ │ ♡  [Свободно 15–22 июля]       │ │  ← бейдж доступности + иконка избранного
│ ├─────────────────────────────────┤ │
│ │ Студия у моря, 2 этаж           │ │  ← название, 1 строка
│ │ 🏖 7 мин пешком · 👤 2 места   │ │  ← расстояние до моря + вместимость
│ │ ❄️ 🅿️ 📶                        │ │  ← иконки: кондей, парковка, WiFi
│ │ 4 500 ₽ / ночь  →  31 500 ₽ ⓘ │ │  ← цена/ночь + итог за период
│ └─────────────────────────────────┘ │
│                                     │
│ [следующая карточка...]             │
└─────────────────────────────────────┘
```

**Элементы:**
- Sticky-шапка: параметры поиска (даты + гости) + кнопка «Изменить» + счётчик + «Фильтры»
- Карточка квартиры:
  - Фото-карусель (2–3 фото, горизонтальный свайп внутри карточки)
  - Бейдж доступности поверх фото: «Свободно 15–22 июля» (зелёный)
  - Иконка ♡ (избранное) — правый верхний угол фото
  - Название / тип квартиры
  - Расстояние до моря **в минутах** (не метрах)
  - Вместимость
  - Иконки удобств: кондиционер, парковка, WiFi (только топ-3)
  - Цена за ночь крупно + итоговая сумма за выбранный период (серым, меньше)
- Скелетон-заглушки при загрузке (не спиннер)
- Пустое состояние: «Нет свободных квартир на эти даты» + кнопка «Изменить даты»
- Состояние ошибки сети: иллюстрация + «Нет соединения» + кнопка «Попробовать снова»

**Действия:**
- Тап на карточку → **Экран 4** (детальная страница)
- Тап на фото внутри карточки → свайп по фото (НЕ открывает детальную)
- Тап «Изменить» в шапке → **Экран 1** (смена дат, с текущими значениями)
- Тап «Фильтры» → **Экран 3** (фильтры, bottom sheet)
- Тап ♡ → добавить в избранное (локально, без уведомлений в V1)
- Скролл вниз → lazy load следующих карточек

**Технические детали:**
- Восстанавливать позицию скролла при возврате с детальной страницы
- Lazy load изображений: показывать `thumbnail` (400px WebP) в каталоге
- `HapticFeedback.impactOccurred('light')` при тапе ♡

### Экран 9 — Оплата (Telegram Payments)

**Что видит пользователь:**
- Нативный sheet Telegram с деталями платежа
- Введение карты или выбор сохранённой
- Кнопка «Оплатить» от Telegram (не наш интерфейс)

**Технические детали:**
```javascript
WebApp.openInvoice(invoiceUrl, (status) => {
  if (status === 'paid') {
    showLoadingScreen(); // НЕ считать подтверждённым здесь!
    startPollingBookingStatus(holdToken, 2000, 30000);
    // Реальное подтверждение придёт через bot webhook
  }
  if (status === 'cancelled') {
    releaseHold(holdToken);
    // Вернуть на Экран 8 с тостом «Оплата отменена»
  }
});
```

**Переходы:**
- Оплата успешна + подтверждение от сервера → **Экран 10** (успех)
- Отмена оплаты → **Экран 8** (шаг 3, тост «Можно попробовать снова»)
- Ошибка оплаты → **Экран 8** (тост «Оплата не прошла», `HapticFeedback.notificationOccurred('error')`)

## Чего НЕ будет в первой версии

| Функция | Причина | Когда |
|---|---|---|
| Карта (Map view) | Требует Leaflet + геолокация + сложная UX | Сезон 2 |
| Избранное с уведомлениями | Требует отдельный bot onboarding flow — без него уведомления невозможны | Сезон 2 |
| Система отзывов | Нет истории аренды на старте, фейковые отзывы опасны | Сезон 2 |
| Реферальная программа | Усложняет MVP, нужна база пользователей | Сезон 2 |
| AI-поиск на естественном языке | Сложная интеграция, нет нужды до 50+ квартир | Сезон 3 |
| Кабинет владельца | Отдельная роль, отдельный UX — не для MVP | Сезон 2 |
| Мультиязычность (EN, BG) | Только русский в V1, контент не готов | Сезон 1 |
| Сохранённые карты / рекуррентные платежи | Telegram Payments не поддерживает без Stars | Сезон 2 |
| Синхронизация с Avito / Sutochno | Интеграция с внешними API | Сезон 2 |

## Требования к дизайну

| Параметр | Требование |
|---|---|
| Цветовая тема | Только `--tg-theme-*` CSS-переменные, никакого хардкода |
| Минимальный размер кнопки | 44×44px (все интерактивные элементы) |
| Шрифт | Системный (`font-family: system-ui`) |
| Анимации | Только `transform` и `opacity`. Никаких `width/height/top/left` анимаций |
| Язык | Русский |
| Стиль | Нативный Telegram — минимум декора, максимум контента |
| Фото в каталоге | WebP, thumbnail 400px для списка, полный размер по требованию |
| Белая вспышка | Устранена через `background_color` в манифесте + `ready()` первой строкой |
